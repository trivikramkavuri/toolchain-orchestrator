# Contributing

Thank you for considering a contribution to `toolchain-orchestrator`.

## Bug reports

Please open an issue with:

- A concise, descriptive title.
- Clear steps to reproduce the problem.
- The actual and expected behaviour.
- Your Node.js version, operating system, and any relevant logs.

## Feature requests

If you would like a new feature, open an issue describing:

- The problem you are trying to solve.
- How the feature fits into the existing pipeline and rule model.
- Any prior art or examples from other tools.

## Pull requests

1. Fork the repository.
2. Create a feature branch: `git checkout -b feat/short-description`.
3. Make your changes and add or update tests where appropriate.
4. Run `npm test` and ensure the test suite passes.
5. Push to your fork and open a pull request.

Please keep pull requests focused and self-contained. If you are planning a larger change, open an issue first so we can discuss the design.
